# GCIP Tagger — Phase 1

The expert **Human-in-the-Loop tagging interface** for the Goal Creation Intelligence
Platform. **We only record goals** — each logged sequence isolates the decisive
goal-creation chain **Pre-Assist → Assist → Goal** with standardized spatial coordinates
and the real World Cup 2026 players involved.

Branded with the Calcio AC and FIFA World Cup 2026 visual identity. The CAC logo links to
[calcioac.com](https://calcioac.com).

## Run

```bash
cd tagger
npm install
npm run dev      # opens http://localhost:5180
```

## Workflow

1. **Match Setup (step 1)** — pick the two countries (home / away) from the official
   World Cup 2026 squad lists, plus competition / date / tagger.
2. **Video Source (step 2)** — paste a YouTube match URL (or bare video id) and hit *Load*.
3. **Tag Action (step 3)** — for each of the three actions (Pre-Assist / Assist / Goal),
   choose the **country** (one of the two teams), the **player** from the filtered squad
   dropdown, and the **body part** (right foot / left foot / head / other), then **arm** the
   action (`P` / `A` / `G` or click the row).
4. **Click the pitch** to plot the armed action. Coordinates are normalized to a
   **105 × 68 m** pitch; the match clock + video timestamp are captured automatically.
   Tags **auto-advance** the chain (Pre-Assist → Assist → Goal). Press `N` for a new goal.
   - **Goal = two clicks:** first the **shot start** (where it was struck), then the
     **shot end location** (where the ball finished). The trajectory is drawn on the pitch.
     Press `Esc` to skip the end click.
5. **Push to Supabase** — see below. (Work also auto-saves to `localStorage`.)

### Keyboard shortcuts

| Key | Action |
|-----|--------|
| `P` / `A` / `G` | Arm Pre-Assist / Assist / Goal |
| `N` | New goal sequence |
| `Esc` | Disarm tag |
| `Space` | Play / pause |
| `←` / `→` | Step one frame (hold `Shift` = ±5s) |

## Squad data

`public/assets/squads.json` is extracted from the official `SquadLists-English.pdf`
(48 teams × 26 players). Re-generate it any time with the parser in the project notes;
each player carries `number`, `pos`, and `name` (name-on-shirt).

## Data model (export shape)

```jsonc
{
  "schemaVersion": 1,
  "match":     { "id", "videoUrl", "videoId", "competition",
                 "homeTeam", "homeCode", "awayTeam", "awayCode", "matchDate", "tagger" },
  "sequences": [ { "id", "matchId", "index", "outcome": "goal" } ],
  "events":    [ { "id", "sequenceId", "type", "minute", "second", "videoTime",
                   "x", "y", "endX", "endY", "player", "teamCode", "bodyPart" } ]
}
```

`event.type ∈ { pre_assist, assist, goal }`,
`event.bodyPart ∈ { right_foot, left_foot, head, other }`,
`events.x ∈ [0,105]`, `events.y ∈ [0,68]` (`endX`/`endY` = shot end location, goals only)
(meters). This maps cleanly onto the PostgreSQL `matches` / `attack_sequences` / `events`
tables for the Phase 2 backend.

Work is auto-saved to `localStorage`; use **Export** for a portable file and **Import**
to resume.

## Supabase sync (optional)

The **⬆ Push to Supabase** button uploads the current project (match + goals + events)
to your Supabase database as a new match snapshot.

1. Copy `.env.example` → `.env` and fill in `VITE_SUPABASE_URL` and
   `VITE_SUPABASE_ANON_KEY` (then restart `npm run dev`).
2. Apply the SQL in `supabase/` in order: `01_schema.sql`, `02_seed_squads.sql`, and
   `03_policies_anon.sql` (the last one lets the browser write with the anon key).

The button only appears when `.env` is configured. Each push inserts a **new** match row
(re-syncing creates another snapshot); IDs are generated by Postgres.
